Research Projects

BikewaySim: Using cycling routing to systematically plan new cycling infrastructure

BikewaySim is a tool intended for helping cities, DOTs, MPOs, or advocacy groups evaluate potential cycling infrastructure/networks. It relies on cycling impedance to find the optimal route from point A to point B according to travel time + a cyclists’ preferences for road attributes (e.g., road gradient, number of lanes, car traffic volume, bicycle infrastructure separated out by type, type of intersection (signalized, two way stop with cross traffic that doesn’t stop), etc.). Cycling impedance is calibrated using GPS traces from cyclists which might be collected through an app/travel diary/etc. Using cycling impedance, the optimal routes between a set of origins and destinations (from TDM OD matrices or travel diaries) can be used to calculate the total cycling impedance for each trip. Then, the process is repeated with a different network with new cycling infrastructure to see how that infrastructure reduced cycling impedance. This reduction can be used to prioritize new infrastructure accordingly.

BikewaySim is a collection of Python, Jupyter Notebook, and Shell/Batch scripts for:

  1. Processing OpenStreetMap (OSM) data into an all viable paths and routable cycling network using OSMnx, Geofabrik, and Osmium
  2. Conflating and reconciling attributes from other network sources to OSM
  3. Map matching cycling GPS traces to OSM using Leuven Map Matching
  4. Calibrating custom link + turn impedance functions (i.e., edge + turn costs) for Dijkstra shortest path routing using Stochopy
  5. Evaluating new cycling infrastructure based on change in aggregate impedance using NetworkX

Follow its development on GitHub.

BikewaySim + TransitSim for Modeling Bike + Transit Trips

Public transit in the United States often requires trekking long distances at the start and/or end of a trip, commonly referred to as the first-last mile problem. Bicycles are one possible solution to this problem that have seen widespread use in the Netherlands, where ~40% of passenger rail trips begin by bike. In the United States however, these trips are not well catered to and trip planning apps often don't provide bike + transit directions.

This project involved modeling bike + transit trips in the Metropolitan Atlanta Rapid Transportation Authority's (MARTA) service area. The bicycle portion of trips was modeled using Dijkstra's algorithm on OSM data, and the transit portion of trips was modeled using the RAPTOR algorithm implemented in the Transit-Routing Python library.
